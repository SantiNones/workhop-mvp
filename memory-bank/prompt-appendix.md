## Prompt 1 — MVP Product Planner
**Prompt**
Create a concise MVP product planner for a two-sided app named “Workhop” where customers can reserve seats/time slots in cafés to work and café owners can define availability and view reservations. Keep scope strict: customer flow (list → detail → select slot → book → confirmation), owner flow (define slots, view reservations). Explicitly list out-of-scope items: payments, full authentication, complex backend logic. Include success metrics: reserve CTA clicks, completed bookings, drop-offs.

## Prompt 2 — Requirements document
**Prompt**
Turn the Workhop MVP planner into a build-ready MVP requirements document. Include personas, user stories, functional requirements grouped by feature, acceptance criteria for each major requirement, non-functional requirements, a lightweight data model (entities + fields), and key edge cases. Keep it aligned to strict MVP scope and explicitly call out non-goals.

## Prompt 3 — Scope check
**Prompt**
Review the Workhop MVP requirements and suggest any missing acceptance criteria or edge cases, but do not expand scope beyond the strict MVP. If you propose anything non-MVP, put it under “Post-MVP ideas.”
